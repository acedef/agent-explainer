Self-Contained Adversary Simulation

Agent executes pre-defined playbooks that replicate real-world attack techniques. The shellcode contains all of the agent's tasks and executes them autonomously without operator interaction, resulting in repeatable, fire-and-forget adversary simulation.

Because the agent is entirely self-contained, it propagates through the environment via injection or lateral movement by replicating itself and updating only the task byte. Each replica knows exactly which task set to execute, enabling complex multi-stage attack chains from a single deployment.

1
Constants — Reusable data like BOF payloads, C2 domains, file paths, and credentials defined once and referenced by index throughout the playbook.
2
Task Sets — Pre-defined agent behaviors identified by numeric IDs. Each task set contains a sequence of operations the agent will execute.
3
Opcodes — The agent supports a variety of opcodes for offensive operations: privilege escalation, process injection, lateral movement, credential access, and more.
4
Embedded Python — Full Python scripting support including standard libraries and direct access to all agent opcodes via the agent module.
02 Core Capabilities
▶

Playbook-Driven Simulation

Define attack sequences in human-readable JSON, compile to compact bytecode, and execute controlled simulations that mirror actual threat actor behavior with precision timing. AI-friendly format makes playbook generation simple.

◎

Rich Technique Coverage

Process injection, credential access, lateral movement, persistence mechanisms, registry manipulation, Active Directory attacks, token manipulation, and more. Cobalt Strike BOF support for any techniques not covered natively.

λ

Embedded Python Runtime

Extend simulations with Python scripts for custom logic, data transformation, or integration workflows. Full access to standard libraries and all agent opcodes via the agent module.

◇

Minimal Footprint

Compiles to position-independent shellcode with minimal footprint. Can be deployed—and deploy itself—as an executable, DLL, or service executable for maximum flexibility.

⚠ Current Limitations

No EDR evasion built-in currently. The COFF loader is functional but not stealthy. This can be addressed quickly if needed. Custom loaders can be developed and behavior extended via Python scripts or BOFs. Additional opcodes planned for future development.

03 Technical Architecture

Engineering Highlights

▣

Position-Independent Code

Compiles to shellcode that can execute from any memory location. Ideal for injection testing and memory-resident simulation scenarios.

⊕

Opcode Architecture

Each technique is an opcode in the bytecode VM. New techniques can be developed and added quickly if needed, or implemented directly in a Python script or BOF.
